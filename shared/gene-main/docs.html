Introduction

Welcome to the official documentation for GENOMA. This project is an experimental, generative content system built on a powerful metaphor: digital genetics. Instead of writing a static document, you assemble a genome from a sequence of codons. This genome acts as a dynamic blueprint that can be expressed to create a rich, interactive phenotype (the final HTML output).

This approach cleanly separates the underlying data structure from its visual presentation, enabling complex behaviors like mutation and recombination. This guide provides a comprehensive overview of the core concepts, the information ontology (codons), and the advanced features of the GENOMA editor.

Core Concepts

The Genome: Digital DNA

The genome is the central data structure in GENOMA. It is an ordered list of codons, stored as a JSON object. Think of it as the digital DNA for your content—it contains all the information needed to construct the final output, but it is not the output itself. This abstraction is what makes advanced genetic operations possible.

The Codon: Unit of Information

A codon is the smallest functional unit in a genome. Each codon is an object with a specific type (e.g., TXT for text, IMG for image), a payload containing its data, and some metadata. Codons are the building blocks you manipulate in the editor to construct the phenotype.

The Phenotype: Expressed Form

The phenotype is the visible, rendered output of a genome. It is the result of the expression process. In technical terms, it's the HTML document generated by the GENOMA engine based on the genome's sequence of codons.

Expression: From Code to Content

Expression is the one-way process of translating a genome into its phenotype. When you click the "Express" button, the engine iterates through the genome's active codons (exons) and renders the corresponding HTML elements in the output panel. This allows for a powerful separation of concerns between data and presentation.

Codon Reference

Core & Media

The fundamental building blocks for content and media embedding.

TXT Text Block

The most fundamental codon. It holds a string of plain text that will be rendered as a paragraph.

Example Payload:

Any string of text, like "The future is layered."

HDR Header

Used for creating section titles. The payload is the text of the header.

Example Payload:

A string for the title, e.g., "Chapter 1: Introduction".

LNK Hyperlink

Creates a hyperlink. The payload should be a valid URL.

Example Payload:

A full URL, like "https://en.wikipedia.org/wiki/Information_theory".

IMG Image

Embeds an image. The payload can be a URL or a base64-encoded data URI.

Example Payload:

A URL: "https://picsum.photos/seed/future/400/200"
Or a Data URI: "data:image/png;base64,iVBORw..."

VID Video

Embeds a video. The payload should be a URL to a video file.

Example Payload:

A URL to a video file, e.g., "/media/intro.mp4".

AUC Audio Clip

Embeds an audio file. The payload should be a URL to an audio file.

Example Payload:

A URL to an audio file, e.g., "/media/soundscape.mp3".

Functional & Genetic

Codons that apply styles, run code, and manage genetic expression.

STY Style Block

Injects a block of CSS into the phenotype for custom styling.

Example Payload:

A string of valid CSS code, e.g., "body { background: #111; }"

CDE Code Snippet

Injects a block of JavaScript into the phenotype for custom interactivity.

Example Payload:

A string of valid JavaScript, e.g., "document.body.addEventListener('click', () => console.log('Body clicked!'));"

BRK Break

Inserts a thematic break or horizontal rule (`<hr>`). It has no payload.

Example Payload:

None required.

CMT Commentary

An editor-only note, not visible in the final output.

Example Payload:

Any string of text, e.g., "TODO: Find a better image."

EXON Expressed Codon

An active, expressed codon included in the final phenotype. This is the default state.

Example Payload:

This is a state, not a type. Any codon can be an exon.

INTRON Unexpressed Codon

A dormant codon ignored during expression but saved in the genome.

Example Payload:

This is a state, not a type. Any codon can be an intron.

Structural & Layout

Codons for creating structural containers like rows and columns. These are essential for building complex, responsive layouts. All structural codons must be properly nested and closed with an `END` codon.

ROW Row Start

Begins a horizontal flex container (a row). All `COL` codons should be placed inside a `ROW`. It can also accept a payload for custom CSS styles (e.g., `gap: 30px;`).

COL Column Start

Begins a vertical container (a column) inside a `ROW`. It can accept a payload to define its size (e.g., `flex: 2;`) or other CSS styles.

END Structure End

Closes the most recently opened `ROW` or `COL` container. Proper nesting is crucial.

Editor Guide

This guide explains how to use the editor's interface and powerful features to create, modify, and evolve your digital genomes.

The Interface

The editor is split into two main panels:

  • Genome Editor (Left): This is your primary workspace. It displays the sequence of codons that make up the currently loaded genome.
  • Expression Engine (Right): This panel shows the rendered phenotype after you click "Express". It is a live preview of what your genetic code produces.

Basic Operations

Adding a Codon

To add a new codon, hover over the space between existing codons. A (+) button will appear. Clicking it opens the Codon Palette, where you can select the codon type and enter its payload. For media codons (IMG, VID, AUC), you will be prompted to select a file directly.

Editing a Codon

Simply click on a codon's payload text in the Genome Editor to make it editable. Press Enter or click outside the input field to save your changes.

Advanced Features & FAQ

GENOMA includes several powerful features for advanced users, allowing for genetic manipulation, phenotype capture, and more.

Capture Phenotype

This feature, powered by the html2canvas library, allows you to export the current rendered phenotype as a self-contained HTML file. It captures the exact visual output, including all styles and layout, and packages it for easy sharing or archiving. Click the Capture Phenotype button in the main controls to use it.

Example: Recombination

Recombination creates a new genome by combining two parent strands at a specified crossover point. The new genome takes the beginning of Strand A and the end of Strand B.

Before:

Strand A: [TXT] [IMG] [STY] [ROW] [END]
Strand B: [HDR] [VID] [TXT] [CDE] [END]

After Recombination:

New Genome: [TXT] [IMG] [VID] [TXT] [CDE] [END]
Example: Mutation Types

Mutations introduce random changes into a genome. Here are the three types:

Point Mutation (Changes a codon)
Before: [TXT] [IMG] [STY]
After: [TXT] [VID] [STY]
Insertion (Duplicates a codon)
Before: [TXT] [IMG] [STY]
After: [TXT] [IMG] [IMG] [STY]
Deletion (Removes a codon)
Before: [TXT] [IMG] [STY]
After: [TXT] [STY]

Frequently Asked Questions

Q: Is my genome saved automatically?

A: Yes, the last active genome is automatically saved to your browser's local storage. This ensures your work is not lost between sessions.

Q: Can I share a genome with someone else?

A: Yes. Use the Export button to save the active genome as a `.json` file. You can then send this file to someone else, and they can use the Import button to load it into their editor.
